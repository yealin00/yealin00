### 💡 오늘 배운 것
오늘은 스터디를 통해 JS의 주요 개념들을 한 번에 묶어서 복습하며 정리했다.
특히 함수의 동작 방식과 스코프, 그리고 데이터 구조인 Map과 Set을 다시 다뤄보며 동작 원리를 더 정확히 이해하려고 했다.

✔함수 관련 개념
- Destructuring(구조분해)
배열이나 객체에서 원하는 값만 꺼내 쓰는 문법. 코드가 깔끔해지고 변수 선언이 쉬워진다.
- Rest Parameter(...)
정해지지 않은 개수의 값을 배열 형태로 받는 방법. 함수 인자를 유연하게 처리할 수 있다.
- Default Parameter
함수 매개변수에 기본값을 지정해 예외적인 상황에서도 안정적으로 동작하도록 한다.
- Scope(스코프)
변수의 유효 범위. let, const, var의 차이를 다시 정리하면서 블록 스코프와 함수 스코프의 차이를 정확히 이해했다.
- Callback Function
다른 함수의 인자로 전달되어 실행되는 함수. 비동기 작업이나 순차 실행할 때 자주 사용된다.
- Hoisting(호이스팅)
변수와 함수 선언이 코드 최상단으로 끌어올려지는 것처럼 동작하는 JS의 특징. var는 선언과 초기화가 함께 호이스팅되어 주의가 필요하다.
- Recursion(재귀함수)
함수가 자기 자신을 호출하는 방식. 종료 조건이 없으면 무한 반복되므로 항상 base case가 필요하다는 걸 배웠다.
- Closure(클로저)
함수가 선언될 당시의 외부 스코프를 기억하고 접근할 수 있는 특징. 데이터 은닉이나 상태 유지에 유용하다.
- Constructor(생성자 함수)
new 키워드와 함께 사용하며 객체를 만드는 함수. 공통 구조를 가진 여러 객체를 생성할 때 사용한다.
✔ Map & Set
- Map
key–value 구조를 가지며 객체보다 더 자유로운 키 설정이 가능. 반복문으로 돌리기 편하고, 데이터 저장/검색에 명확하다.
- Set
중복을 허용하지 않는 데이터 집합. 배열의 중복 제거나 빠른 검색에 유용하다.

Map/Set 두 개 모두 기존 배열보다 더 구조적인 데이터 처리 방식이라는 점을 확인했다.
