### 💡 오늘 배운 것
1. 프로그래머스 코딩테스트 — 입력 받는 구조 이해하기
오늘은 프로그래머스에서 문자열을 출력하는 아주 기본적인 문제를 풀었는데,
입력을 받는 방식 때문에 혼란이 컸다.
✔ 내가 겪은 혼란
- readline / readFileSync 차이
- 모듈(require)이 뭔지
- Node.js가 뭔지 모르는데 코드가 그걸 쓰고 있음
- 실행은 안 되고, 에러도 안 뜨고, “한 글자도 못 적겠다”는 상태 도달
✔ 해결한 내용
- 프로그래머스는 readline 방식이 거의 안 됨
- Node.js 환경에서 JavaScript가 실행되고 있었던 것

2. Node.js / 모듈의 느낌만 이해하기
아직 배운 건 아니지만, 코테 환경에서 자연스럽게 마주쳤다.
- Node.js → 브라우저 밖에서 돌아가는 JavaScript
- 모듈(require) → 필요한 기능을 불러오는 도구 세트
- fs, readline 같은 것들이 전부 "입출력 담당 도구"

3. 사소하지만 중요한 배움
오늘 코드를 통해 배운 생짜 기초:
- 입력은 line 이벤트로 들어온다
- close 이벤트 안에서 출력하면 된다
- input 배열 첫 번째 값이 str이 된다
- 괄호 하나만 빠져도 코드 전체가 죽는다
- 언어 설정이 C면 JS 코드를 아무리 넣어도 실행이 안 된다 (!!)
