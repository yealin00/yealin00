### 💡 오늘 배운 것
1. 프론트엔드 → 백엔드 → 파파고 API 흐름 이해
- 사용자가 “안녕”이라고 입력하면 프론트엔드(내 웹페이지)에서 그 값을 백엔드로 보내고,
- 백엔드 서버가 네이버 파파고 API에 요청을 날려서 “Hello!”라는 번역 결과를 받아오고,
- 그걸 다시 프론트엔드로 보내서 사용자에게 출력!

오늘의 핵심은…
“내 웹 → 내 서버 → 네이버 서버 → 내 서버 → 내 웹”
이런 구조를 처음으로 직접 경험했다는 점!

2. 서버가 왜 필요한지 처음으로 감 잡음
- API Key 같은 중요한 값은 프론트에 두면 위험함 → 백엔드에서 처리해야 함
- 프론트엔드는 사용자와 대화하고, 백엔드는 외부 서비스와 대화한다는 걸 이해함

3. HTTP 요청과 응답의 흐름 이해
- 프론트 → 백엔드로 fetch
- 백엔드 → 파파고로 request
- 파파고 → 백엔드로 response
- 백엔드 → 프론트로 다시 response
→ “데이터가 이렇게 왔다 갔다 하는구나!!” 감이 조금씩 잡힘

4. 동기/비동기가 왜 필요한지도 감이 옴
- 번역 결과가 바로 오는 게 아니라 “기다렸다가” 오는 구조라서 비동기 코드(async/await)가 나온다는 걸 알게됨.
