# 2025.12.05 🌸
오늘은 JavaScript가 실제 웹페이지와 어떻게 연결되는지 이해하기 위해
JSON 데이터 처리와 DOM(Document Object Model) 개념을 집중적으로 공부했다.
특히 데이터를 불러와서 화면에 반영하는 과정이 어떻게 이루어지는지 감을 잡는 데 시간을 많이 썼다.

### 💡 오늘 배운 것
- JSON의 구조와 역할
JSON은 서버와 클라이언트가 데이터를 주고받을 때 사용하는 형식이고,
문자열을 JavaScript 객체로 바꿀 때 JSON.parse(),
반대로 객체를 문자열로 바꿀 때 JSON.stringify()를 사용한다는 걸 배웠다.
직접 문제를 풀면서 JSON 안의 값을 어떻게 꺼내오는지 연습했다.
- 객체 내부 데이터 접근 방식
data.user.name, skills[1]처럼
점 표기법과 배열 인덱스를 혼합해서 데이터를 읽는 방식을 여러 번 연습했다.
복잡한 JSON 구조도 차근차근 따라가면 읽을 수 있다는 자신감이 조금 생겼다.
- DOM(Document Object Model)
HTML 문서가 JavaScript에서 ‘트리 구조’처럼 보인다는 개념을 처음으로 이해했다.
웹페이지의 태그 하나하나가 객체가 되고,
자바스크립트는 이 객체들을 조작해 화면을 바꾼다는 흐름이 잡혔다.
- DOM 선택자 사용법
getElementById, querySelector, querySelectorAll을 이용해 원하는 요소를 찾는 방법을 연습했다.
콘솔에서 직접 가져온 요소의 textContent를 출력해보면서
내가 선택한 요소가 맞는지 확인하는 과정이 재미있었다.

### 🧠 느낀 점
처음에는 JSON도 낯설고 DOM도 막연했는데,
직접 코드를 찍어보고 데이터를 꺼내보면서 하나씩 이해되기 시작했다.
특히 “브라우저가 HTML을 DOM으로 바꾸고,
JavaScript가 그 DOM을 조작해 화면이 바뀌는 구조”가 잡히니까
웹 개발이 어떻게 동작하는지 흐름이 보이기 시작했다.
