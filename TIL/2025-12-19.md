# 2025.12.19 🌸
오늘은 캐러셀 구현을 중심으로 프론트엔드와 서버를 연결하는 흐름을 경험한 하루였다.
처음에는 이미지 3장을 HTML에 직접 넣고 버튼으로 좌우 이동하는 기본 캐러셀을 만들었다.
코드 리뷰에서 “이미지를 직접 넣기보다는, 데이터를 가져와서 화면을 만들면 더 좋겠다”
라는 피드백을 받았는데, 이 말이 실무에서는 데이터가 먼저이고 화면은 그 결과라는 의미라는 걸 알게 됐다.
그래서 HTML에는 구조만 남기고, JS에서 데이터 배열을 읽어 slide를 동적으로 생성하는 방식으로 생각을 전환했다.
이 과정에서 “JS가 화면을 만든다”는 개념이 조금 더 명확해졌다.또한 Live Server가 아닌,
Node + Express 서버를 직접 띄워서 터미널로 실행하는 과정도 진행했다.
- npm init
- npm install express
- 서버 파일 실행 (node server.mjs)
- localhost:3000으로 접속

중간에 package.json, 실행 경로, ES module 설정 때문에 꽤 헤맸지만
그만큼 서버는 ‘파일을 여는 것’이 아니라 ‘프로그램을 실행하는 것’이라는 차이를 확실히 느꼈다.
오늘은 정신적으로도 쉽진 않았지만, 결과적으로는 프론트엔드에서 데이터 → 화면 → 서버로 이어지는 큰 그림을 처음으로 잡은 날이었다.

💡 오늘 배운 것
- 캐러셀은 index + translateX로 구현할 수 있다
- HTML에 직접 이미지 넣는 것보다 데이터 기반 렌더링이 더 실무적이다
- 서버는 반드시 올바른 경로에서 실행해야 한다
- Live Server와 Node 서버는 역할이 완전히 다르다
- JS는 “보조”가 아니라 화면을 만드는 주체가 될 수 있다
