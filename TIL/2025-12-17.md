💡 오늘 배운 것
1. 터미널 사용에 익숙해짐
 - cd로 폴더 이동하기
 - node 파일이름으로 JS 파일 실행하기
2. API 엔드포인트와 HTTP 메서드
  클라이언트가 서버에 요청하는 방법은 크게 두 가지
  - GET : 조회
  - POST : 추가 / 수정 / 처리 요청
API 엔드포인트는 서버가 제공하는 기능의 “주소”라는 개념을 이해했다.
3. API를 사용·테스트·소비하는 방법
 API를 다루는 방식은 크게 3가지가 있다.
 - GUI 방식 (Apidog, Postman 등)
 - CLI 방식
 - 프로그래밍 코드로 직접 요청 (fetch 등)
상황에 따라 적절한 방식을 선택해 테스트할 수 있다는 걸 배움.
4. 번역 API 요청에 필요한 데이터
 - source (원본 언어)
 - target (번역할 언어)
 - text (번역할 텍스트)
→ API 요청에는 반드시 정해진 파라미터 구조가 필요하다는 점을 알게 됐다.
5. 함수와 파라미터, 프로퍼티 개념 정리
 - 함수는 특정 동작을 묶어둔 코드 덩어리
 - 파라미터는 함수가 값을 받기 위해 사용하는 이름
 - 인자는 실제로 전달되는 값
 - 객체에서 name, age 같은 것은 키이자 프로퍼티 이름이라는 점을 정리했다.
6. 정보처리기사 자격증
 - 강사님과 보조강사님 모두 CS 기초를 위해 추천
 - 개발 공부와 병행하면 도움이 될 수 있다는 조언을 들었다.
