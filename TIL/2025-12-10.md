### 💡 오늘 배운 것
1. Nullish 병합 연산자 (??)
오늘 처음 본 문법인데 생각보다 엄청 실용적임! ?? 는 왼쪽 값이 null 또는 undefined일 때만 오른쪽 값을 사용한다.
```
let name = null;
let result = name ?? "이름 없음";  // → "이름 없음"
```
핵심:
0, "", false 같은 값은 “값이 있음”으로 취급함 → 대체값 안 씀
→ OR(||)과 완전 다름!

2. 자바스크립트 에러 핸들링 개요
(try / catch / finally)
에러가 발생하더라도 프로그램이 터지지 않게 하는 구조.
```
try {
  // 위험한 코드 실행 (여기서 에러 날 수도 있음)
} catch (e) {
  // 에러 발생 시 실행되는 구역
} finally {
  // 무조건 실행되는 구역 (선택)
}
```
try → "일단 해볼게!"
catch → "아…망했다. 근데 내가 잡음"
finally → "성공하든 실패하든 난 무조건 일함"

3. sort() — 정렬 (그리고 혼란…)
sort는 사실 JS에서도 유명한 난제 중 하나. 왜냐면 정렬 기준을 내가 직접 정해야 하고, 문자열 기준으로 정렬하는 게 기본이라 숫자 정렬하면 망함.
예)
```
[10, 2, 5].sort(); 
// → [10, 2, 5] 이상한 정렬…
```
제대로 하려면:
```
[10, 2, 5].sort((a, b) => a - b);  
// → [2, 5, 10]
```

✔ 강사님 말:
“이건 다들 헷갈리니까 검색해서 쓰세요 ㅋㅋ”

4. 불변성을 통한 코드 신뢰성
오늘 들은 말 중 가장 현실적인 조언: 원본 데이터는 건드리지 마라. 복사 → 가공 → 사용 원본 훼손하면 진짜 망함.
✔ 왜?
어디서 어떻게 값이 바뀌었는지 추적이 불가능해짐.
✔ 그래서
변하지 않는 값은 const 재할당이 꼭 필요하면 let 가능한 모든 배열/객체는 복사해서 사용
복사 예시:
```
const newArray = [...oldArray];   // 스프레드 → 권장
```
5. 변수 유효 범위(스코프): var, let, const
오늘의 핵심은 “왜 var 쓰면 안 되는가?”
✔ var
함수 스코프
호이스팅 때문에 선언 전에 접근 가능
그래서 예측 불가능해짐
✔ let / const
블록 스코프
선언 전에 접근 불가 (TDZ 때문에 에러 발생)
TDZ(Temporal Dead Zone):
“선언은 되었지만 초기화 전까지 접근할 수 없는 죽음의 구역 ⚠️”

6. 즉시 실행 함수 IIFE + 호이스팅 막기
즉시 실행 함수 형태:
```
(function() {
  // 바로 실행됨
})();
```
이 함수는 스코프를 하나 더 만들어서
var의 이상한 호이스팅 문제를 피할 수 있었지만…
현대 JS는 그냥 let, const 써서 깔끔하게 해결이라는 분위기.

7. 배열 복사 및 정렬 (불변성 핵심)
원본 배열 건드리면 나중에 진짜 혼란 생김.
그래서 반드시 복사 → 정렬 순서로 진행해야 한다.
```
const copied = [...arr];  // 권장
copied.sort();
```
✔ 객체 배열도 마찬가지로 복사해야 함.
✔ 다른 복사 방식도 있지만 스프레드가 깔끔함.

8. 로컬 스토리지 & 메모리 관리
오늘 암기한 핵심 3가지:
✔ 로컬 스토리지(LocalStorage)
브라우저 꺼도 남아 있음
문자열만 저장 가능 → 객체는 JSON.stringify 필요
```
localStorage.setItem(
  'userProfile', 
  JSON.stringify(userProfile)
);
```
✔ 세션 스토리지(SessionStorage)
창 닫으면 날아감
